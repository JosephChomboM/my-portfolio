# Design System

## Overview

This design system is for Joseph Chombo’s AI / ML engineering portfolio.

The visual language should feel calm, organic, intelligent, precise, and editorial. It is inspired by organic systems, koi-like motion, branching structures, natural computation, information flow, and thoughtful technical work.

The website should not look like a generic SaaS landing page, a neon AI startup, or a creative developer portfolio copied from another designer. It should feel like a serious AI engineering portfolio with a distinctive organic visual identity.

The design should communicate:

- AI engineering
- Machine learning systems
- Retrieval and reasoning
- Evaluation and evidence
- MLOps discipline
- Systems thinking
- Technical clarity
- Quiet visual craft

The uploaded visual references should be used as a design language, not as a single image pasted into the website.

## Core Direction

The site should feel like:

Calm.  
Technical.  
Organic.  
Minimal.  
Research-oriented.  
Premium but not flashy.  
Visual, but still readable.  
Systematic, but not cold.

The visual identity should combine organic movement with engineering structure.

The portfolio should prove that Joseph can think in systems, not just decorate a page.

## Core Palette

Use this exact core palette from the uploaded brand board.

- **Deep Ink** `#121212`: Main text, logo, strong lines, borders, dark surfaces, high-contrast details.
- **Vermilion** `#E4572E`: Main signal color. Use for section numbers, small dots, active states, CTAs, and important accents.
- **Clay** `#D9A07A`: Warm secondary accent. Use for organic surfaces, soft highlights, background depth, and calm visual warmth.
- **Sage** `#7A8F72`: System and growth color. Use for AI systems, branching structures, architecture sections, and calm technical elements.
- **Parchment** `#F2EEE6`: Main background color. Use as the default page surface.

The site should be led by Parchment, Deep Ink, Vermilion, Clay, and Sage.

## Accent Colors

Use these exact secondary accent colors from the uploaded brand board.

- **Space Indigo** `#252739`: Use for technical depth, code-related sections, infrastructure, metadata, dark details, and serious engineering emphasis.
- **Taupe** `#907473`: Use for soft human warmth, reflective sections, subtle card accents, and neutral supporting UI.
- **Light Bronze** `#D1985A`: Use for diagram highlights, important nodes, architecture emphasis, and warm signal accents.
- **Blue Slate** `#4D6A72`: Use for data, retrieval, embeddings, system links, chart accents, and cool technical contrast.

Accent colors are supporting colors. They should not dominate the website.

The approximate color usage ratio should be:

70% Parchment and neutral space.  
20% Deep Ink text, lines, and structure.  
8% core accents, mainly Vermilion, Clay, and Sage.  
2% secondary accent colors.

Do not introduce unrelated blues, purples, neon colors, gradients, or cyberpunk tones.

## Color Rules

Use Parchment as the default page background.

Use Deep Ink for typography, logo marks, primary borders, and important interface structure.

Use Vermilion sparingly. Vermilion should behave like a signal, not a background theme.

Use Sage for sections about systems, growth, architecture, process, and AI workflows.

Use Clay for warmth, secondary visual emphasis, soft cards, and organic background shapes.

Use Space Indigo only when the design needs technical depth or a serious dark contrast.

Use Blue Slate for diagrams, data connections, embeddings, retrieval flows, and evaluation visuals.

Avoid pure white unless needed for contrast. Prefer Parchment or warm off-white surfaces.

Avoid heavy shadows. Use thin borders, spacing, opacity, and layering instead.

## Typography

Use clean, modern sans-serif typography.

Preferred font: Inter.

If Inter is unavailable, use Geist, IBM Plex Sans, or a similar clean sans-serif font.

The typography should feel editorial, precise, and technical.

## Headline Style

Headlines should be large, confident, and readable.

Use medium or semi-bold weight.

Avoid overly decorative fonts.

Use tight but readable letter spacing.

Headlines should feel serious and calm, not playful.

Example headline:

“Building intelligent systems with context, evaluation, and deployment discipline.”

Another acceptable headline:

“AI engineering shaped by retrieval, evidence, and systems thinking.”

## Body Text

Body text should be clear and direct.

Use 15–18px body text.

Use generous line height, around 1.6 to 1.8.

Avoid vague marketing language.

Prefer concrete engineering language: retrieval, ranking, evaluation, evidence, deployment, observability, model behavior, architecture, validation.

## Labels

Use uppercase labels with wide letter spacing.

Labels should be used for section numbers, metadata, navigation, stack names, and case study fields.

Examples:

`01 / FLOW MOVEMENT`  
`03 / ORGANIC INTELLIGENCE`  
`AI / ML ENGINEERING PORTFOLIO`  
`TYPE`  
`STACK`  
`IMPACT`

Labels should usually use Deep Ink or Vermilion.

## Layout Principles

Use an editorial grid.

Prefer spacious layouts with strong hierarchy.

Use thin dividing lines inspired by the uploaded board.

The layout can feel like a refined technical poster, but it must still function as a website.

Use wide margins.

Use section numbers.

Use project metadata.

Use visual motifs as part of the structure.

Avoid cramped dashboards.

Avoid generic centered landing-page sections.

Avoid too many cards with equal visual weight.

The portfolio should guide the user through a clear story:

Hero.  
Selected work.  
AI engineering approach.  
System architecture.  
Case studies.  
Technical stack.  
About.  
Contact.

## Visual Motif System

The uploaded design board contains eight motif families. Use them as reusable website components.

Use this motif for the homepage hero, section transitions, and subtle background motion.

Visual qualities:

Thin flowing curves.  
Koi-like organic marks.  
Sparse dots.  
Soft directional movement.  
Rhythm and calm motion.  
Light linework over Parchment.

Meaning:

Flow of ideas.  
Search through information.  
Movement through systems.  
Learning and adaptation.  
Navigation across projects.

Recommended website use:

Hero background.  
Top-page animated flow.  
Section dividers.  
Footer linework.  
Page transition motif.

Curves should move slowly.  
Organic marks can drift along curved paths.  
Dots can fade in and out.

## 05 / Pattern Tiles

Use pattern tiles as background textures and surface treatments.

Visual qualities:

Dense rhythmic patterns.  
Sparse minimal patterns.  
Repeated koi-like marks.  
Light paper-like texture.  
Optional dark inverse versions.

Recommended website use:

Card backgrounds.  
Footer backgrounds.  
Hover states.  
Project thumbnails.  
Section textures.  
Dark mode panels.  
Case study separators.

Rules:

Use low opacity behind content.  
Do not place dense patterns behind important body text.  
Use sparse patterns for readable sections.  
Use dense patterns only in decorative areas or thumbnails.  
Use inverse patterns only for dark sections.

Pattern tiles should make the site feel alive without hurting readability.

## 06 / Color Variants

Use the logo and mark variants intentionally.

Logo states:

Ink: Deep focus.  
Vermilion: Signal and energy.  
Sage: Balance and growth.  
Clay: Grounded warmth.  
Ink inverse: Clarity and depth.  
Sage inverse: Calm and renewal.

Recommended use:

Use the Deep Ink logo on Parchment by default.

Use the Vermilion logo only for active states, special moments, or small brand accents.

Use the Sage logo for systems, architecture, and process sections.

Use the Clay logo for warm, reflective, or about sections.

Use the inverse logo only on dark surfaces.

Do not show too many logo color variants in the same view.

## 07 / Palette

Use the palette as global design tokens.

The whole website should be built from the core palette and accent colors listed above.

Do not invent additional colors unless absolutely necessary for code syntax, charts, or accessibility.

If charts are needed, use Deep Ink, Vermilion, Sage, Clay, Space Indigo, Light Bronze, and Blue Slate.

## 08 / Alternate Directions

Use alternate directions for page-specific variation.

Recommended mapping:

AI / RAG pages: Organic Intelligence motif.  
MLOps / backend pages: Branching Systems motif.  
Data validation pages: Data Connections motif.  
Homepage: Flow Movement motif.  
About page: Sparse Pattern Tiles.  
Dark mode or special sections: Ink Inverse pattern.  
Warm reflective sections: Clay or Taupe accents.  
Technical diagrams: Blue Slate or Space Indigo accents.

Alternate directions should create variation without breaking the identity.

## Buttons

Buttons should be minimal.

Primary button:

Deep Ink background.  
Parchment text.  
Medium rounded corners.  
No heavy shadow.  
Small Vermilion dot or line on hover.

Secondary button:

Transparent background.  
Deep Ink border.  
Deep Ink text.  
Soft background fill on hover.

Button corners should use 12px or 24px depending on size.

Avoid glossy effects, gradients, and exaggerated hover animations.

## Cards

Cards should rely on structure, border, and spacing.

Default card style:

Parchment or slightly lighter warm surface.  
1px border using `#D8D0C4`.  
Rounded corners between 12px and 24px.  
No heavy shadow.  
Optional subtle motif background at 5–10% opacity.

Cards should feel editorial, not like SaaS dashboard cards.

## Motion Principles

Motion should be subtle, slow, and purposeful.

Use motion to show:

Flow.  
Retrieval.  
Reasoning loops.  
System behavior.  
Connection.  
Feedback.

Recommended motion:

Slow drifting organic marks.  
Dots gently fading in and out.  
Thin lines drawing on scroll.  
Soft project card reveals.  
Very slow orbit rotation.  
Subtle page transitions.  
Small hover movement.

Motion timings:

Small hover: 150–250ms.  
Section reveal: 500–800ms.  
Background drift: 8–20s.  
Orbit rotation: 24–60s.  
Line drawing: 800–1400ms.

Use smooth easing.

Avoid:

Fast animations.  
Bouncy easing.  
Aggressive scroll effects.  
Heavy 3D.  
Random particle systems.  
Excessive parallax.  
Motion behind dense text.  
Animations that reduce readability.

## Interaction Style

Interactions should be quiet and precise.

Hover states can use:

Small Vermilion dot.  
Thin line reveal.  
Slight opacity shift.  
Soft background change.  
Gentle card lift.

Avoid:

Large scaling.  
Rotating cards.  
Neon glows.  
Complex cursor effects.  
Distracting hover animations.

## Accessibility

Maintain readable contrast.

Text over Parchment should usually use Deep Ink.

Avoid placing body text over dense patterns.

Use low-opacity motifs behind text.

Interactive elements should be visually clear.

Do not rely on color alone to communicate state.

Use visible focus states.

Primary text should maintain at least 4.5:1 contrast where possible.

## Do’s

Do use the exact palette from the uploaded board.

Do use Parchment as the main background.

Do use Deep Ink for serious structure and readability.

Do use Vermilion sparingly as a signal.

Do use the visual motifs as a consistent system.

Do make the site feel calm, technical, and distinctive.

Do use project metadata and case study structure.

Do show engineering depth.

Do use motion to explain flow and intelligence.

Do keep the interface readable.

## Don’ts

Do not paste the full moodboard into the website as the design.

Do not copy another portfolio’s identity.

Do not make the site look like a generic AI SaaS landing page.

Do not use neon gradients, glowing brains, robots, cyberpunk graphics, or generic AI visuals.

Do not overuse accent colors.

Do not put dense patterns behind important text.

Do not prioritize motion over content.

Do not make the design so artistic that the engineering work becomes unclear.

Do not add complex WebGL or 3D unless there is a strong reason.

Do not use too many visual motifs in one section.

The final website should feel like a calm AI engineering portfolio with an organic intelligence identity.

Editorial layout.
